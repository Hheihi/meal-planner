# 今日吃什么 — 前端架构设计（Taro + React）

## 1. 技术栈选型

| 类别 | 技术方案 | 版本 | 说明 |
|------|---------|------|------|
| 跨端框架 | Taro | 4.x | 基于 React 语法，编译到微信小程序，未来可扩展 H5/RN |
| UI 框架 | React | 18.x | 函数组件 + Hooks |
| 语言 | TypeScript | 5.x | 全量 TS，提升可维护性 |
| 状态管理 | Zustand | 4.x | 轻量、React Hooks 友好，替代 Redux 减少样板代码 |
| 样式方案 | Less Module | - | 组件级样式隔离，避免全局污染 |
| 请求库 | Taro.request 封装 | - | 统一请求/响应拦截、Token 管理、错误处理 |
| 包管理 | pnpm | - | 速度快、磁盘占用小 |
| 代码规范 | ESLint + Prettier | - | 统一代码风格 |

## 4. 网络层设计（api/request.ts）

### 4.1 请求封装核心逻辑

```
请求发出
  │
  ├── 请求拦截器
  │   ├── 自动注入 Authorization: Bearer <token>
  │   ├── 统一 Content-Type: application/json
  │   └── 附加公共参数（设备信息等，按需）
  │
  ├── Taro.request 发送请求
  │
  └── 响应拦截器
      ├── code === 0 → 返回 data
      ├── code === 1002（Token 失效）→ 自动静默登录 → 重放原请求
      ├── code === 其他业务错误 → reject(BusinessError)
      └── HTTP 异常 → reject(NetworkError)
```

### 4.2 Token 自动续签流程

```
请求返回 code=1002
  │
  ├── 加锁（防止并发续签）
  ├── 调用 Taro.login() 获取新 code
  ├── 调用 POST /v1/auth/login 换取新 token
  ├── 更新本地存储 token
  ├── 释放锁
  └── 重放所有排队中的请求
```

### 4.3 API 模块示例（api/menu.ts）

```typescript
// 每个 API 文件只负责接口定义和调用，不处理业务逻辑
import { get, post, del } from './request'
import type { TodayMenuResponse, AddDishParams } from './types'

export const menuApi = {
  getToday: () => get<TodayMenuResponse>('/v1/menus/today'),
  addDish: (params: AddDishParams) => post('/v1/menus/dishes', params),
  removeDish: (menuItemId: number) => del(`/v1/menus/dishes/${menuItemId}`),
  confirm: (date: string, mealType: number) => post('/v1/menus/confirm', { date, meal_type: mealType }),
  unconfirm: (date: string, mealType: number) => post('/v1/menus/unconfirm', { date, meal_type: mealType }),
}
```

## 5. 状态管理设计（Zustand）

### 5.1 Store 划分原则

| Store | 职责 | 持久化 |
|-------|------|--------|
| `useAuthStore` | token、登录态、登录方法 | token 持久化到 Storage |
| `useUserStore` | 用户信息、口味偏好 | 用户信息缓存到 Storage |
| `useMenuStore` | 今日菜单数据、营养汇总 | 不持久化，每次 onShow 刷新 |
| `useAppStore` | 全局 loading、网络状态 | 不持久化 |

### 5.2 Store 示例（useMenuStore）

```typescript
interface MenuState {
  todayMenu: TodayMenu | null
  loading: boolean
  fetchToday: () => Promise<void>
  addDish: (dishId: number, mealType: MealType) => Promise<void>
  removeDish: (menuItemId: number) => Promise<void>
  confirmMeal: (mealType: MealType) => Promise<void>
}
```

### 5.3 数据流向

```
用户操作 → 页面组件 → 调用 Store Action → Store 调用 API → 更新 Store 状态 → 组件自动重渲染
                                                │
                                          失败时 → Toast 提示，状态不变
```

## 7. 自定义 Hooks 设计

### 7.1 useRequest — 通用请求 Hook

```typescript
function useRequest<T>(
  fetcher: () => Promise<T>,
  options?: {
    manual?: boolean       // true=手动触发, false=组件挂载时自动请求
    onSuccess?: (data: T) => void
    onError?: (err: Error) => void
  }
): {
  data: T | undefined
  loading: boolean
  error: Error | undefined
  run: () => Promise<void>
  refresh: () => Promise<void>
}
```

### 7.2 useAuth — 登录态 Hook

```typescript
function useAuth(): {
  isLoggedIn: boolean
  login: () => Promise<void>          // 静默登录
  ensureLogin: () => Promise<void>    // 确保已登录，未登录则自动登录
}
```

```typescript
function useReachBottom<T>(

  list: T[]
```

## 9. 数据缓存策略

| 数据 | 缓存位置 | 过期策略 | 说明 |
|------|---------|---------|------|
| Token | Taro.setStorageSync | 7 天（由后端控制） | 过期自动静默续签 |
| 用户信息 | Store + Storage | onLaunch 时从 Storage 恢复，接口刷新覆盖 | 减少首屏白屏 |
| 今日菜单 | Store（不持久化） | 每次 onShow 重新请求 | 保证数据实时性 |
| 菜品详情 | 内存 Map | 5 分钟 | 减少短时间重复请求 |
| 菜系列表 | Store + Storage | 24 小时 | 低频变更数据 |
| 推荐列表 | Store（不持久化） | 不缓存，每次请求 | 保证推荐新鲜度 |

## 11. 错误处理与用户反馈

### 11.1 错误分层

```
网络层错误（无网络 / 超时 / 500）
  └── request.ts 拦截 → Taro.showToast('网络异常，请稍后重试')

业务层错误（code !== 0）
  ├── 1002 Token 失效 → 自动静默续签（用户无感知）
  ├── 2001 资源不存在 → 页面内 EmptyState 展示
  ├── 2002 重复操作 → Toast 提示（如"该菜品已在菜单中"）
  └── 其他 → Toast 展示 message 字段

页面层错误（首屏加载失败）
  └── ErrorRetry 组件，提供重试按钮
```

### 11.2 Loading 状态规范

| 场景 | 方案 |
|------|------|
| 首屏数据加载 | Skeleton 骨架屏 |
| 操作反馈（加入菜单、确认完成） | Taro.showLoading + Taro.hideLoading |
| 列表分页加载 | 底部 loading 指示器 |
| 下拉刷新 | 小程序原生下拉刷新 |

## 12. 性能优化策略

| 策略 | 说明 |
|------|------|
| 图片懒加载 | 列表页使用 Taro `<Image>` 的 `lazyLoad` 属性 |
| 分包加载 | 非 TabBar 页面按模块拆分为子包，减少首包体积 |
| 骨架屏 | 首屏/列表页使用骨架屏替代白屏等待 |
| 请求并行 | 首页 onShow 并行请求菜单概览 + 推荐列表 |
| 本地缓存 | 菜系列表/菜品详情短期缓存，减少重复请求 |
| 防抖 | 搜索输入防抖 300ms，减少无效请求 |
| 虚拟列表 | 推荐列表若数据量增大，使用 `@tarojs/components` 虚拟列表 |

## 13. 类型系统设计

### 13.1 通用响应类型（types/common.ts）

```typescript
// 统一 API 响应结构
interface ApiResponse<T = unknown> {
  code: number
  message: string
  data: T
}

  page_size: number
```
