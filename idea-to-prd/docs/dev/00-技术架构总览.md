# 今日吃什么 — 技术架构总览

## 1. 技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端 | 微信小程序原生框架（WXML + WXSS + JS/TS） | 官方框架，兼容性最佳，也可选用 Taro/uni-app 跨端框架 |
| 后端 | Node.js (Koa/Express) 或 Go / Python (FastAPI) | 根据团队技术栈选择 |
| 数据库 | MySQL / PostgreSQL | 关系型数据，适合结构化菜品和用户数据 |
| 缓存 | Redis | 热门推荐缓存、用户会话管理 |
| 对象存储 | 腾讯云 COS / 阿里云 OSS | 菜品图片、步骤图片存储 |
| 部署 | 微信云开发 或 自建服务器 | 云开发可降低运维成本 |

## 3. 核心数据模型总览

### 3.1 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK | 用户ID |
| openid | VARCHAR(64), UNIQUE | 微信 openid |
| unionid | VARCHAR(64) | 微信 unionid（可选） |
| nickname | VARCHAR(64) | 昵称 |
| avatar_url | VARCHAR(256) | 头像URL |
| taste_preferences | JSON | 口味偏好标签，如 `["偏辣","清淡"]` |
| family_group_id | BIGINT, FK, NULLABLE | 所属家庭群组 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 菜系表 (cuisines)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT, PK | 菜系ID |
| name | VARCHAR(32) | 菜系名称，如"川菜" |
| icon_url | VARCHAR(256) | 菜系图标 |
| sort_order | INT | 排序权重 |

### 3.3 菜品表 (dishes)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK | 菜品ID |
| cuisine_id | INT, FK | 所属菜系 |
| name | VARCHAR(64) | 菜品名称 |
| image_url | VARCHAR(256) | 菜品主图 |
| ingredients | JSON | 食材清单，如 `[{"name":"豆腐","amount":"1块"}]` |
| steps | JSON | 制作步骤，如 `[{"order":1,"text":"...","image_url":"..."}]` |
| nutrition | JSON | 营养信息 `{"calories":350,"protein":15,"carbs":20,"fat":12}` |
| tags | JSON | 标签，如 `["辣","快手菜"]` |
| difficulty | TINYINT | 难度 1-3 |
| cooking_time | INT | 烹饪时间（分钟） |
| status | TINYINT | 状态：1=上架 0=下架 |
| created_at | DATETIME | 创建时间 |

### 3.4 每日菜单表 (daily_menus)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK | 记录ID |
| user_id | BIGINT, FK | 用户ID |
| date | DATE | 日期 |
| meal_type | TINYINT | 餐次：1=早餐 2=午餐 3=晚餐 |
| dish_id | BIGINT, FK | 菜品ID |
| is_confirmed | BOOLEAN | 是否已确认完成 |
| created_at | DATETIME | 创建时间 |

**复合唯一索引**：`(user_id, date, meal_type, dish_id)`

### 3.5 饮食记录表 (diet_records)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK | 记录ID |
| user_id | BIGINT, FK | 用户ID |
| date | DATE | 日期 |
| meal_type | TINYINT | 餐次 |
| record_type | TINYINT | 类型：1=菜单自动生成 2=手动文字记录 |
| dish_id | BIGINT, FK, NULLABLE | 菜品ID（自动记录时） |
| manual_text | VARCHAR(256) | 手动文字记录内容 |
| nutrition_snapshot | JSON | 营养快照（记录时刻的营养数据） |
| created_at | DATETIME | 创建时间 |

### 3.6 家庭群组表 (family_groups)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK | 群组ID |
| name | VARCHAR(64) | 群组名称 |
| owner_id | BIGINT, FK | 创建者用户ID |
| invite_code | VARCHAR(32), UNIQUE | 邀请码 |
| created_at | DATETIME | 创建时间 |

### 3.7 家庭群组成员表 (family_group_members)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK | 记录ID |
| group_id | BIGINT, FK | 群组ID |
| user_id | BIGINT, FK, UNIQUE | 用户ID（每人只能加入一个群组） |
| role | TINYINT | 角色：1=创建者 2=普通成员 |
| joined_at | DATETIME | 加入时间 |

## 4. 模块依赖关系

```
用户系统（基础模块，其他所有模块依赖）
  │
  ├── 菜品库（独立模块，仅依赖基础数据）
  │     │
  │     ├── 智能推荐（依赖菜品库 + 饮食记录）
  │     │
  │     └── 今日菜单（依赖菜品库）
  │           │
  │           └── 饮食记录（依赖今日菜单的确认完成）
  │
  └── 家庭群组（依赖用户系统 + 今日菜单）
```

## 5. API 规范约定

### 5.1 基础约定

- **Base URL**: `https://api.example.com/v1`
- **协议**: HTTPS
- **数据格式**: JSON
- **认证方式**: 请求头携带 `Authorization: Bearer <token>`，token 由微信登录后服务端签发

### 5.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

错误码约定：

| code | 说明 |
|------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | 未登录 / token 失效 |
| 1003 | 无权限 |
| 2001 | 资源不存在 |
| 2002 | 重复操作 |
| 5000 | 服务器内部错误 |

### 5.3 分页约定

```json
{
  "code": 0,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

## 6. 小程序页面路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/pages/index/index` | 首页 | 推荐 + 今日菜单概览 |
| `/pages/cuisine/list` | 菜系列表 | 菜系分类浏览 |
| `/pages/cuisine/dishes` | 菜品列表 | 某菜系下菜品列表 |
| `/pages/dish/detail` | 菜品详情 | 菜谱、做法、营养 |
| `/pages/menu/today` | 今日菜单 | 按餐管理菜单 |
| `/pages/record/index` | 饮食记录 | 日历 + 历史记录 |
| `/pages/record/detail` | 记录详情 | 某日饮食详情 |
| `/pages/family/index` | 家庭群组 | 群组管理 |
| `/pages/family/member-menu` | 成员菜单 | 查看成员今日菜单 |
| `/pages/profile/index` | 个人中心 | 设置与入口 |
| `/pages/profile/preferences` | 偏好设置 | 口味偏好 |
| `/pages/search/index` | 搜索 | 菜品搜索 |

## 7. 开发建议的模块拆分顺序

1. **Phase 1**: 用户系统 → 菜品库 → 首页与菜品详情页
2. **Phase 2**: 今日菜单 → 饮食记录
3. **Phase 3**: 智能推荐 → 家庭群组
4. **Phase 4**: 联调测试 → 性能优化 → 上线
