# 模块七：首页与菜品详情页

## 1. 模块概述

首页是用户打开小程序的第一个页面，承载今日菜单概览和智能推荐两个核心功能区域。菜品详情页是各入口共用的菜品信息展示页面。本文档定义首页的整体布局和菜品详情页的完整交互。

## 2. 功能清单

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| H-01 | 首页整体布局 | P0 | TabBar 首页 |
| H-02 | 今日菜单概览区 | P0 | 首页顶部，快速了解今日用餐安排 |
| H-03 | 智能推荐区 | P0 | 首页下半部，推荐菜品卡片流 |
| H-04 | 搜索入口 | P0 | 首页顶部搜索框 |
| H-05 | 菜品详情页 | P0 | 完整的菜品信息展示和加入菜单操作 |

### 3.2 TabBar 配置

| 位置 | 标签 | 图标 | 路径 |

| 1 | 首页 | home | `/pages/index/index` |

### 3.3 搜索入口 (H-04)

**交互**：
- 首页顶部显示搜索框占位区域（不可直接输入）
- 点击后跳转到独立搜索页 `/pages/search/index`
- 搜索页支持实时搜索（详见菜品库模块 D-04）

### 3.4 今日菜单概览区 (H-02)

**数据来源**：调用 `GET /v1/menus/today` 接口

**展示规则**：
- 三个餐次横排展示，等宽分布
- 每个餐次卡片显示：
  - 餐次标签（早餐/午餐/晚餐）
  - 第一道菜品名称（超过 1 道显示"N道菜"）
  - 已确认状态 ✅
  - 未安排显示"暂未安排"
- 点击单个餐次卡片 → 跳转今日菜单页
- 右上角"查看全部" → 跳转今日菜单页

**加载状态**：
- 数据加载中：显示骨架屏
- 加载失败：显示简化版本"点击查看今日菜单"

### 3.5 智能推荐区 (H-03)

**数据来源**：调用 `GET /v1/recommendations` 接口

**推荐卡片设计**：
```
┌──────────────────────────────┐
│ [菜品大图 750×300]            │
│                               │
│  菜品名称         菜系名称   │
│  🔥 350kcal  ⏱ 25min  ⭐简单 │
│                               │
│               [ + 加入菜单 ]  │
└──────────────────────────────┘
```

**交互**：
- 推荐卡片上下滑动浏览
- 点击卡片 → 进入菜品详情页
- 点击"加入菜单" → 弹出餐次选择 ActionSheet
- 点击"换一批" → 调用刷新推荐接口，列表更新（加载动画）
- 下拉刷新 → 刷新整个首页（菜单概览 + 推荐列表）

**加载状态**：
- 首次加载：骨架屏（3 个卡片占位）
- 换一批：卡片区域显示加载动画
- 加载失败：显示"加载失败，点击重试"

### 4.3 底部操作栏

**固定在页面底部**，不随页面滚动。

**方案 A — 三按钮**（推荐）：
```
┌──────────┬──────────┬──────────┐
│ + 早餐    │ + 午餐    │ + 晚餐   │
└──────────┴──────────┴──────────┘
```
- 直接点击对应按钮加入该餐
- 已添加到该餐的按钮变灰显示"已添加"

**方案 B — 单按钮 + 弹窗**：
```
┌────────────────────────────────┐
│         [ + 加入菜单 ]          │
└────────────────────────────────┘
```
- 点击后弹出 ActionSheet 选择餐次

### 4.4 交互细节

**图片查看**：
- 菜品大图支持点击全屏预览
- 步骤图支持点击放大查看

**加入菜单反馈**：
- 成功：Toast "已加入午餐菜单" + 对应按钮变灰
- 重复：Toast "该菜品已在午餐菜单中"
- 失败：Toast "添加失败，请重试"

**页面来源适配**：
- 从首页推荐进入：正常显示
- 从菜系浏览进入：正常显示
- 从搜索结果进入：正常显示
- 从饮食记录进入：正常显示（可加入菜单）
- 从家庭成员菜单进入：正常显示（可加入自己的菜单）

## 5. 前端组件拆分建议

### 5.1 公共组件

| 组件 | 用途 |
|------|------|
| `dish-card` | 菜品卡片（推荐列表、搜索结果、菜品列表通用） |
| `meal-selector` | 餐次选择 ActionSheet |
| `nutrition-bar` | 营养信息横排展示组件 |
| `nutrition-summary` | 营养汇总卡片 |
| `empty-state` | 空状态占位组件 |
| `skeleton` | 骨架屏组件 |

### 5.2 dish-card 组件

**Props**：
```javascript
{
  dish: {
    id: Number,
    name: String,
    image_url: String,
    cuisine_name: String,
    cooking_time: Number,
    difficulty: Number,
    calories: Number
  },
  showAddButton: Boolean,  // 是否显示加入菜单按钮
  cardStyle: String        // 'large' | 'small' 卡片尺寸
}
```

**Events**：
```javascript
{
  onTap: (dishId) => {},          // 点击卡片
  onAddToMenu: (dishId) => {}     // 点击加入菜单
}
```

### 5.3 meal-selector 组件

**Props**：
```javascript
{
  visible: Boolean,
  dishId: Number,
  addedMeals: Array  // 该菜品已添加到的餐次 [1, 2]
}
```

**Events**：
```javascript
{
  onSelect: (mealType) => {},  // 选择餐次
  onClose: () => {}            // 关闭
}
```

## 6. 首页数据加载策略

### 6.1 加载顺序

```
首页 onLoad
  ├── 并行请求
  │   ├── GET /v1/menus/today       → 渲染菜单概览
  │   └── GET /v1/recommendations   → 渲染推荐列表
  └── 两个请求独立渲染，互不阻塞
```

### 6.2 缓存策略

- 今日菜单数据：本地缓存 + 每次 onShow 刷新
- 推荐列表：不缓存（每次 onShow 可选择是否刷新）
- 菜品详情：本地缓存 5 分钟，减少重复请求

### 6.3 下拉刷新

- 下拉刷新同时刷新菜单概览和推荐列表
- 使用 `wx.startPullDownRefresh()` / `wx.stopPullDownRefresh()`

## 7. 边界与异常处理

| 场景 | 处理方式 |
|------|---------|
| 首页数据全部加载失败 | 显示全局错误页，提供重试按钮 |
| 菜单概览加载失败但推荐正常 | 菜单概览区域显示"加载失败"，推荐正常展示 |
| 菜品详情页传入无效 dish_id | 显示"菜品不存在"提示，引导返回 |
| 图片加载失败 | 显示默认占位图 |
| 加入菜单时网络中断 | Toast 提示失败，保持按钮可点击 |
| 用户未登录时的首页展示 | 静默登录（wx.login），不影响页面加载 |
