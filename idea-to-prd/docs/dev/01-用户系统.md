# 模块一：用户系统

## 1. 模块概述

用户系统是整个小程序的基础模块，负责用户认证、信息管理和口味偏好设置。所有其他模块都依赖用户系统提供的身份识别能力。

## 2. 功能清单

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| U-01 | 微信一键登录 | P0 | 小程序启动时静默获取 openid，需要用户信息时引导授权 |
| U-02 | 获取/更新用户信息 | P0 | 昵称、头像 |
| U-03 | 口味偏好设置 | P1 | 可选设置，用于辅助推荐 |
| U-04 | 个人中心页面 | P1 | 各功能入口聚合 |

## 3. 详细需求

### 3.1 微信登录 (U-01)

**流程描述**：
1. 用户打开小程序 → 前端调用 `wx.login()` 获取临时 `code`
2. 前端将 `code` 发送到后端 `/auth/login` 接口
3. 后端使用 `code` 向微信服务器换取 `openid` 和 `session_key`
4. 后端检查该 `openid` 是否已注册：
   - 已注册：更新 `session_key`，返回自定义 token
   - 未注册：创建新用户记录，返回自定义 token
5. 前端将 token 存储在本地 `wx.setStorageSync('token', token)`
6. 后续请求通过 `Authorization` 请求头携带 token

**Token 管理**：
- Token 有效期：7 天
- Token 过期后，前端重新调用 `wx.login()` 静默续签
- 无需用户手动操作

**业务规则**：
- 首次进入无需弹出授权弹窗，静默登录即可
- 当需要展示用户昵称/头像时（如个人中心、家庭群组），引导用户点击按钮授权
- 使用微信小程序最新的 `<button open-type="chooseAvatar">` 和昵称输入组件获取用户信息

### 3.2 用户信息获取/更新 (U-02)

**获取用户信息**：
- 通过微信授权获取昵称和头像
- 用户可在个人中心修改昵称

**业务规则**：
- 未授权用户显示默认头像和"微信用户"昵称
- 授权后信息存入数据库

### 3.3 口味偏好设置 (U-03)

**偏好标签列表**（预定义，后台可配置）：

| 分类 | 标签选项 |
|------|---------|
| 口味 | 偏辣、清淡、偏甜、偏酸、重口味 |
| 饮食方式 | 素食、低碳水、高蛋白、无辣 |
| 忌口 | 不吃牛肉、不吃海鲜、不吃内脏、乳糖不耐、花生过敏 |

**交互方式**：
- 以标签多选的形式展示
- 用户可随时进入个人中心修改偏好
- 偏好设置为可选，不设置不影响基础功能使用

**业务规则**：
- 偏好数据存入用户表的 `taste_preferences` 字段
- 推荐模块读取偏好数据调整推荐结果
- 设置忌口标签后，推荐结果应排除对应菜品

## 5. API 接口

### 5.1 微信登录

```
POST /v1/auth/login
```

**请求参数**：
```json
{
  "code": "wx_login_code_string"
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "token": "jwt_token_string",
    "is_new_user": true,
    "user": {
      "id": 10001,
      "nickname": "微信用户",
      "avatar_url": "",
      "taste_preferences": [],
      "family_group_id": null
    }
  }
}
```

### 5.2 获取当前用户信息

```
GET /v1/user/profile
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "id": 10001,
    "nickname": "小明",
    "avatar_url": "https://...",
    "taste_preferences": ["偏辣", "高蛋白"],
    "family_group_id": 1,
    "family_group_name": "小明的家"
  }
}
```

### 5.3 更新用户信息

```
PUT /v1/user/profile
```

**请求参数**：
```json
{
  "nickname": "新昵称",
  "avatar_url": "https://..."
}
```

### 5.4 更新口味偏好

```
PUT /v1/user/preferences
```

**请求参数**：
```json
{
  "taste_preferences": ["偏辣", "高蛋白", "不吃海鲜"]
}
```

## 6. 边界与异常处理

| 场景 | 处理方式 |
|------|---------|
| 微信 `wx.login` 调用失败 | 提示网络异常，引导用户重试 |
| 后端登录接口返回失败 | 提示登录失败，提供重试按钮 |
| Token 过期 | 自动静默重新登录，不中断用户操作 |
| 用户拒绝授权昵称/头像 | 使用默认值，不影响核心功能 |
| 偏好保存失败 | 提示保存失败，保留用户当前选择，支持重试 |
