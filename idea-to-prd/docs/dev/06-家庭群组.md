# 模块六：家庭群组

## 1. 模块概述

家庭群组模块支持用户创建家庭，通过微信分享邀请家人加入，成员之间可共享查看今日菜单。本模块仅提供只读共享能力，不支持协作编辑。

## 2. 功能清单

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F-01 | 创建家庭群组 | P0 | 用户创建群组并命名 |
| F-02 | 生成邀请链接 | P0 | 通过微信分享小程序卡片邀请 |
| F-03 | 加入群组 | P0 | 通过邀请链接加入 |
| F-04 | 查看成员今日菜单 | P0 | 核心功能，共享查看 |
| F-05 | 成员管理 | P1 | 创建者移除成员，成员退出群组 |
| F-06 | 解散群组 | P2 | 创建者解散整个群组 |

## 3. 详细需求

### 3.1 创建家庭群组 (F-01)

**入口**：个人中心 → 家庭群组管理

**流程**：
1. 用户点击"创建家庭"
2. 输入家庭名称（如"小明的家"）
3. 提交创建
4. 自动生成唯一邀请码
5. 用户成为群组创建者（`role=1`）

**业务规则**：
- 每个用户最多加入 1 个群组（创建也算加入）
- 已加入群组的用户不可再创建新群组
- 群组名称不超过 20 字

### 3.2 生成邀请链接 (F-02)

**流程**：
1. 群组创建者/成员点击"邀请家人"
2. 调用微信分享 API，生成小程序分享卡片
3. 分享卡片包含群组邀请码信息
4. 被邀请人点击卡片打开小程序，自动跳转加入流程

**分享卡片内容**：
- 标题："xxx邀请你加入「小明的家」"
- 描述："一起看看今天吃什么吧"
- 缩略图：小程序 logo 或自定义图片

**技术方案**：
```javascript
wx.shareAppMessage({
  title: `${nickname}邀请你加入「${groupName}」`,
  path: `/pages/family/join?invite_code=${inviteCode}`,
  imageUrl: '/images/share-family.png'
})
```

### 3.3 加入群组 (F-03)

**流程**：
1. 被邀请人点击分享卡片打开小程序
2. 小程序解析路径中的 `invite_code`
3. 展示加入确认页面：
   ```
   ┌────────────────────────┐
   │  xxx 邀请你加入         │
   │  「小明的家」           │
   │                         │
   │  当前成员: 2人          │
   │                         │
   │  [ 加入家庭 ]           │
   │  [ 暂不加入 ]           │
   └────────────────────────┘
   ```
4. 确认加入 → 调用加入接口
5. 加入成功 → 跳转到家庭群组页

**业务规则**：
- 用户已在其他群组 → 提示"你已加入其他家庭，需先退出当前家庭"
- 邀请码无效/群组已解散 → 提示"邀请已失效"
- 用户已在该群组 → 提示"你已是该家庭成员"
- 群组成员上限：建议 10 人（可配置）

### 3.5 成员管理 (F-05)

**创建者权限**：
- 查看成员列表
- 移除普通成员
- 移除时弹出确认"确定将小红移出家庭？"

**普通成员权限**：
- 查看成员列表
- 退出群组（自己退出）
- 退出时确认"确定退出「小明的家」？"

**成员管理页**：
```
┌───────────────────────────────┐
│  家庭成员                      │
│                                │
│  [头像] 小明     群主          │
│  [头像] 小红     [移除]       │
│  [头像] 小刚     [移除]       │
│                                │
│  [邀请新成员]                  │
│                                │
│  ─────────────────────────    │
│  [解散家庭]  (仅群主可见)     │
└───────────────────────────────┘
```

### 3.6 解散群组 (F-06)

**仅创建者可操作**：
1. 点击"解散家庭"
2. 弹出二次确认"解散后所有成员将被移除，确定解散？"
3. 确认后：
   - 所有成员的 `family_group_id` 设为 null
   - 删除 `family_group_members` 所有记录
   - 标记群组为已解散
4. 所有成员下次打开群组页时提示"家庭已解散"

## 4. API 接口

### 4.1 创建群组

```
POST /v1/family-groups
```

**请求参数**：
```json
{
  "name": "小明的家"
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "小明的家",
    "invite_code": "ABC123XY",
    "member_count": 1
  }
}
```

### 4.2 获取群组信息

```
GET /v1/family-groups/mine
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "小明的家",
    "invite_code": "ABC123XY",
    "owner": {"id": 10001, "nickname": "小明", "avatar_url": "..."},
    "members": [
      {"id": 10001, "nickname": "小明", "avatar_url": "...", "role": 1},
      {"id": 10002, "nickname": "小红", "avatar_url": "...", "role": 2}
    ],
    "member_count": 2
  }
}
```

**未加入群组时**：
```json
{
  "code": 0,
  "data": null
}
```

### 4.3 通过邀请码获取群组预览

```
GET /v1/family-groups/preview?invite_code=ABC123XY
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "小明的家",
    "member_count": 2,
    "inviter_nickname": "小明"
  }
}
```

### 4.4 加入群组

```
POST /v1/family-groups/join
```

**请求参数**：
```json
{
  "invite_code": "ABC123XY"
}
```

### 4.5 退出群组

```
POST /v1/family-groups/leave
```

### 4.6 移除成员（仅创建者）

```
DELETE /v1/family-groups/members/:user_id
```

### 4.7 解散群组（仅创建者）

```
DELETE /v1/family-groups/mine
```

### 4.8 获取成员今日菜单

```
GET /v1/family-groups/members/:user_id/menu/today
```

**响应格式**：同今日菜单接口（`GET /v1/menus/today`），但返回的是指定成员的菜单。

**权限校验**：仅限同一群组成员访问。

## 5. 边界与异常处理

| 场景 | 处理方式 |
|------|---------|
| 用户已在群组，尝试创建新群组 | 提示"你已加入一个家庭，需先退出" |
| 邀请码过期/无效 | 提示"邀请已失效，请联系家人重新邀请" |
| 群组满员（10人） | 提示"家庭成员已达上限" |
| 创建者退出群组 | 不允许创建者退出，需先解散或转让（V1 不做转让，只能解散） |
| 成员被移除后查看群组页 | 提示"你已不在该家庭中" |
| 查看成员菜单时该成员已退出 | 不再显示该成员，列表实时更新 |
| 网络异常导致加入失败 | 提示重试 |
