# Plan 07：首页布局与搜索页

> **阶段**：Phase 1 — 核心浏览
> **前置依赖**：Plan 02（请求层）、Plan 03（Skeleton / EmptyState）、Plan 04（useDebounce / useReachBottom）
> **产出目标**：首页整体框架搭建（搜索入口 + 菜单概览占位 + 推荐区占位），搜索页功能闭环

---

## 1. 任务清单

### 1.1 首页框架（TabBar 页）

- [ ] 页面路径：`src/pages/index/`
- [ ] 页面配置：`navigationBarTitleText: '今日吃什么'`, `enablePullDownRefresh: true`
- [ ] **页面结构**（从上到下）：
  1. **搜索入口**：搜索框占位区域（不可直接输入），点击跳转搜索页
  2. **今日菜单概览区**：占位区域，显示三个餐次卡片骨架（Plan 09 实现数据对接）
  3. **推荐区域**：占位区域，显示推荐卡片骨架（Plan 10 实现数据对接）
- [ ] 此阶段只搭建布局框架，推荐和菜单概览的数据对接在后续 Plan 中完成

### 1.2 搜索页

- [ ] 页面路径：`src/pages/search/index/`
- [ ] 页面配置：`navigationBarTitleText: '搜索菜品'`
- [ ] 功能实现：
  - 顶部搜索框，自动聚焦，支持实时输入
  - 使用 `useDebounce(keyword, 300)` 防抖
  - 防抖后的关键词调用 `dishApi.search(keyword, page)` 搜索
  - 搜索结果使用列表展示（与菜品列表页复用样式：左图右文）
  - 支持分页加载（`useReachBottom`）
  - 点击搜索结果 → 跳转菜品详情页
- [ ] 状态处理：
  - 搜索框为空 → 显示引导文字"输入菜品名称搜索"
  - 搜索中 → Skeleton（list 类型）
  - 无结果 → EmptyState（"未找到相关菜品，换个关键词试试"）
  - 搜索失败 → ErrorRetry

## 3. 验收标准

1. 首页三个区域（搜索 + 菜单概览 + 推荐）布局正确
2. 点击搜索框跳转到搜索页
3. 搜索页输入关键词后 300ms 防抖触发搜索
4. 搜索结果正确展示，支持分页和触底加载
5. 点击搜索结果跳转菜品详情页
6. 空/无结果/错误状态展示正确
