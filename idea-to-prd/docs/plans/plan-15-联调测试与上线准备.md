## 1. 任务清单

### 1.1 前后端联调

- [ ] 切换 API 地址到测试环境，逐模块联调：
  - [ ] **用户系统**：微信登录 → Token 获取 → 静默续签 → 用户信息授权 → 偏好保存
  - [ ] **菜品库**：菜系列表 → 菜品列表分页 → 菜品详情 → 搜索
  - [ ] **今日菜单**：添加菜品 → 移除菜品 → 营养汇总 → 确认完成 → 取消确认
  - [ ] **智能推荐**：推荐列表 → 换一批 → 加入菜单
  - [ ] **饮食记录**：日历标记 → 日详情 → 手动记录 → 删除记录
  - [ ] **家庭群组**：创建 → 分享邀请 → 加入 → 查看成员菜单 → 移除成员 → 退出 → 解散
- [ ] 记录并修复所有联调问题

### 1.2 核心流程走查

完整走通以下用户路径，确保无阻断：

- [ ] **新用户首次使用**：
  打开小程序 → 静默登录 → 首页展示 → 浏览推荐 → 点击菜品查看详情 → 加入午餐 → 切换到菜单 Tab 查看 → 确认完成 → 切换到饮食记录查看
- [ ] **搜索加菜流程**：
  首页搜索框 → 搜索页输入 → 选择结果 → 详情页 → 加入菜单
- [ ] **偏好设置流程**：
  我的 → 口味偏好 → 选择标签 → 保存 → 返回首页 → 推荐结果应匹配偏好
- [ ] **家庭群组流程**：
  创建家庭 → 分享邀请给好友 → 好友点击加入 → 双方可互看今日菜单
- [ ] **跨天使用**：
  确认今日菜单 → 次日打开 → 菜单为空 → 饮食记录有昨天数据

### 1.3 边界场景测试

- [ ] 弱网络 / 无网络情况下各页面表现
- [ ] Token 过期后自动续签是否中断用户操作
- [ ] 快速连续点击按钮（添加/删除/确认）
- [ ] 超长菜品名称 / 超长食材列表的展示
- [ ] 菜品被下架后访问详情页
- [ ] 群组被解散后成员再次打开
- [ ] 分包页面首次访问的加载速度
